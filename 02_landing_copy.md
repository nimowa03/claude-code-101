## Section 2: 수익화 초입 가이드

Claude Code 101 스타일. 스크롤하면서 읽는 인터랙티브 가이드.
진행률 바가 스크롤에 따라 "Step 1/5" → "Step 5/5 ✓" 업데이트.

### 섹션 타이틀

```
수익화 초입 가이드
— 만든 다음, 뭘 해야 하는지 5단계로 정리했습니다.
```

### 짧은 설명

```
구독으로 받을지, 건당으로 받을지, 무료로 풀고 나중에 받을지.
수익화 구조를 모르면, 좋은 서비스도 돈이 안 됩니다.

읽고 나면, 내 서비스의 첫 스텝이 뭔지 보입니다.
```

---

### Step 1: 뭘 만들었는지 한 줄로 말할 수 있는가

```
[STEP 01]

만든 건 있는데,
"이게 뭐냐"고 물으면 설명이 길어지나요?

한 줄로 못 말하면, 파는 건 더 어렵습니다.

"누구를 위한, 무슨 문제를 해결하는 서비스"
이 한 줄이 수익화의 출발점입니다.

예시:
× "AI 기반 다기능 생산성 플랫폼"
○ "프리랜서가 견적서를 5분 만에 만드는 도구"

💡 설명이 길어질수록, 돈을 내는 사람은 줄어듭니다.
```

---

### Step 2: 누가 돈을 내는가

```
[STEP 02]

"좋은 서비스인데 왜 안 팔리지?"
대부분 여기서 막힙니다.

사용하는 사람과 돈을 내는 사람이 다를 수 있습니다.

개인이 쓰는 도구 → 개인이 직접 결제 (가격에 민감)
회사에서 쓰는 도구 → 회사가 결제 (가치에 민감)
무료로 쓰는 사람 → 직접 안 냄 (광고주가 냄)

"누가 지갑을 여는가"를 정하면,
가격도, 채널도, 메시지도 같이 정해집니다.

💡 "모두를 위한 서비스"는 아무도 돈을 안 냅니다.
   타겟을 좁힐수록 매출은 올라갑니다.
```

---

### Step 3: 어떤 구조로 돈을 받을 것인가

```
[STEP 03]

이제 "어떻게 받을지"를 정할 차례입니다.

구조는 크게 5가지입니다.
내 서비스에 맞는 걸 하나만 고르세요.

① 구독: 매달 결제. 매일 쓰는 도구에 적합.
② 건당/크레딧: 쓴 만큼 결제. AI 서비스에 적합.
③ 프리미엄: 무료로 쓰게 하고 고급 기능에 과금.
④ 수동 먼저: 사람이 먼저 해보고, 되면 자동화.
⑤ 무료 도구→유료 전환: 무료로 트래픽 모으고 천천히 전환.

고르는 기준:
- 유저가 매일 쓰나? → ①
- API 비용이 크나? → ②
- 빨리 유저를 모아야 하나? → ③
- 아직 이게 되는지 모르겠나? → ④
- 검색에서 찾을 수 있나? → ⑤

💡 처음부터 완벽한 모델을 고를 필요 없습니다.
   하나를 골라서 시작하고, 안 되면 바꾸면 됩니다.
```

---

### Step 4: 가격은 어떻게 정하는가

```
[STEP 04]

"얼마를 받아야 하지?"
가장 많이 막히는 지점입니다.

정답은 없지만, 시작점은 있습니다.

방법 1: 비슷한 서비스 가격을 찾아본다
→ 경쟁자가 $10/월이면, $8~15 사이에서 시작

방법 2: "이 문제를 안 풀면 얼마나 손해인가"로 역산
→ 매달 10시간 아끼면, 시급 × 10 = 절약 금액
→ 그 중 10%만 받아도 충분

방법 3: 일단 작게 시작하고 올린다
→ 처음엔 싸게. 피드백 받으면서 가격 올리기.
→ 가격을 올렸는데 이탈이 없으면, 원래 그 가격이 맞는 것.

💡 무료로 시작하면 유료로 바꾸기가 어렵습니다.
   처음부터 아무리 작아도 가격을 붙이세요.
```

---

### Step 5: 첫 번째 고객은 어디서 오는가

```
[STEP 05]

서비스도 있고, 가격도 정했는데.
"어디서 사람이 오지?"

광고 전에, 먼저 해볼 수 있는 것들:

① 커뮤니티에 직접 가서 보여주기
→ 긱어스, IndieHackers, Reddit, 오픈카톡
→ "이거 만들었는데 피드백 좀 주세요"
→ 첫 유저는 광고가 아니라 대화에서 옵니다

② 만든 과정을 공개하기
→ Threads, 블로그, 유튜브
→ 과정 자체가 콘텐츠이자 마케팅

③ 무료 도구 하나를 먼저 공개하기
→ 핵심 기능 중 하나를 무료로 풀기
→ 써본 사람이 유료로 넘어감

💡 "마케팅을 해야 하는데"가 아니라
   "내 서비스가 필요한 사람이 어디에 모여있는지"를 찾으세요.
```

### 가이드 마무리

```
여기까지 읽었으면,
적어도 이건 정리됐을 겁니다:

"내 서비스를 한 줄로 뭐라고 할지"
"누가 돈을 내는지"
"어떤 구조로 받을지"
"얼마를 받을지"
"첫 고객을 어디서 찾을지"

완벽하지 않아도 됩니다.
이 5개가 임시로라도 정해지면, 시작할 수 있습니다.
```

## Section 3: 커뮤니티 + Charlee

### 섹션 타이틀

```
수익화 얘기는 늘 남의 일 같았습니다.
```

### 본문

```
해외 유니콘, 대규모 투자, 팔로워 수십만.
수익화 성공 사례를 볼 때마다
나와는 다른 세상 얘기처럼 느껴집니다.

여기 모이는 사람들은,
진짜 변하고 싶은 사람들입니다.

사이드 프로젝트를 하나 만들어봤는데 수익화가 막막한 사람.
부업으로 서비스를 시작했는데 어디서 돈을 받아야 할지 모르는 사람.
바이브코딩으로 뭔가 만들 줄은 아는데, 파는 건 처음인 사람.

해내고 싶고, 성장하고 싶은 사람들.
멀게만 느껴졌던 수익화 사례들을
우리 상황에 맞게 다시 조립해보는 곳입니다.
```

### Charlee 소개

```
온라인 셀러이자 빌더.
직접 만들고 팔아본 사람이라, 수익화가 어렵고 안 되는 답답함을 압니다.
저도 계속 성장해야 하는 사람이고, 먼저 겪은 시행착오와 찾은 인사이트를 공유합니다.
```
